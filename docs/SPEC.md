# 仕様書: Image-3D — 画像から3Dプリントデータ生成アプリ

## 1. 概要

アップロードされた画像(単一イメージ、またはキャラクターシート等)を元に、
3Dプリンターで使用できるデータ(STL / 3MF / GLB / OBJ)を生成するWebアプリケーション。
生成した3Dモデルはアプリ内の3Dビューアでインタラクティブに確認できる。

## 2. ターゲット環境

| 項目 | 内容 |
|---|---|
| 実行環境 | ローカルマシン(Linux, NVIDIA RTX PRO 6000 Blackwell 96GB VRAM) |
| バックエンド | Python 3.12 / FastAPI / Uvicorn |
| 3D生成モデル | Hunyuan3D-2 (shape generation) を第一候補。プラガブル設計で差し替え可能 |
| メッシュ処理 | trimesh (+ pymeshlab はオプション) |
| フロントエンド | HTML + ES Modules + Three.js(ビルド不要、FastAPIが静的配信) |
| ブラウザ | Chrome / Firefox / Edge 最新版(WebGL2必須) |

## 3. 機能要件

### 3.1 画像アップロード (FR-1)
- PNG / JPEG / WebP をドラッグ&ドロップまたはファイル選択でアップロードできる。
- 最大ファイルサイズ: 20MB。
- キャラクターシート(複数ビューが1枚に並んだ画像)の場合、正面ビューを主入力として使用する
  (マルチビュー入力対応は将来拡張)。
- アップロード画像のプレビューを表示する。

### 3.2 背景除去 (FR-2)
- 生成精度向上のため、アップロード画像から背景を自動除去できる(rembg使用)。
- ON/OFF切り替え可能(デフォルトON)。除去結果をプレビュー表示する。

### 3.3 3Dモデル生成 (FR-3)
- 画像1枚から3Dメッシュを生成する(Image-to-3D)。
- 生成はジョブとして非同期実行し、進捗状態(queued / preprocessing / generating /
  postprocessing / completed / failed)をポーリングAPIで取得できる。
- 生成パラメータ:
  - `steps`: 拡散ステップ数(デフォルト30)
  - `guidance_scale`: ガイダンススケール(デフォルト5.5)
  - `octree_resolution`: メッシュ解像度(256 / 384 / 512、デフォルト384)
  - `seed`: 乱数シード(省略時ランダム)
- ジェネレータはプラガブル(`Generator`インターフェース):
  - `hunyuan3d`: Hunyuan3D-2 shape pipeline(本番用)
  - `mock`: 入力画像に依らず既知メッシュ(例: Utahティーポット/スザンヌ相当)を返す
    開発・テスト用ジェネレータ。モデル未ダウンロード環境でもアプリ全体が動作すること。

### 3.4 3Dプリント向けメッシュ後処理 (FR-4)
生成メッシュに対し以下を自動適用する(trimesh):
- 非多様体・重複面・浮遊小部品(全体積の1%未満の連結成分)の除去
- 穴埋め(watertight化の試行)と watertight 判定結果のレポート
- 法線の統一
- 指定サイズへのスケーリング(目標高さ mm 指定、デフォルト100mm)
- ポリゴン数の簡略化(目標面数指定可、デフォルト200,000面上限)

### 3.5 3Dビューア (FR-5)
- 生成完了したモデルをアプリ内でThree.jsによりレンダリング表示する。
- 操作: 回転(左ドラッグ)/ パン(右ドラッグ)/ ズーム(ホイール)= OrbitControls。
- 表示モード: シェーディング / ワイヤーフレーム切替。
- グリッド床とビルドプレートサイズ目安(220×220mm相当)の表示。
- モデル情報表示: 頂点数・面数・バウンディングボックス寸法(mm)・watertight判定・推定体積(cm³)。

### 3.6 エクスポート (FR-6)
- STL(バイナリ)/ 3MF / GLB / OBJ 形式でダウンロードできる。
- STL/3MFは後処理済み(プリント向け)メッシュを出力する。

### 3.7 4色カラープリント対応 (FR-8)
マルチフィラメント方式のカラー3Dプリンタ(Bambu Lab AMS、Prusa MMU等、最大4色)向けの出力に対応する。

- **カラーモード** (`color_mode`): `none`(単色、デフォルト)/ `color4`(4色カラー)。
- **色付けの方式**: 入力画像(背景除去後)をメッシュ正面軸に沿って直交投影し、
  頂点カラーを取得する(テクスチャ生成AIは使用しない)。正面画像は正面側の頂点に
  のみ投影し、背面画像がある場合は背面側の頂点に背面画像を投影する。背面画像が
  無い場合、背面側と側面/上下の曖昧な頂点はベース色にする。
- **パレット量子化**: 頂点カラーをk-meansで `n_colors`(2〜4、デフォルト4)色に量子化する。
  量子化後のパレット(HEX)と色ごとの面数比率をジョブ`stats.palette`として返す。
- **エクスポート**:
  - 3MF: 色ごとにメッシュを分割し、1ファイル内に最大4オブジェクト
    (名前 `color_1`〜`color_4`、表示色付き)として出力。スライサー側で
    オブジェクトごとにフィラメントを割り当てて印刷する。
  - GLB: 頂点カラー付きで出力し、ビューアでカラー表示する。
  - STL/OBJ: 形状のみ(色情報なし、従来通り)。
- **ビューア**: カラーモード時は頂点カラーを表示。モデル情報バーにパレット
  (色チップ+面数比率)を表示する。
- 色分割された各サブメッシュは後処理(FR-4)適用後のメッシュから生成するため、
  合計形状は単色出力と一致する。サブメッシュ単体はwatertightとは限らない点を
  UIに注記する(積層方式のマルチカラー印刷では通常問題にならない)。

### 3.8 キャラクターシート/マルチビュー入力 (FR-9)【Phase 3】
- 複数ビュー画像(正面必須+背面/左側面/右側面の任意組合せ)をビューラベル付きで
  アップロードでき、複数ビュー時は Hunyuan3D-2mv(multiview shapeモデル)で生成する。
  単一画像時は従来の hunyuan3d-dit-v2-0 を使用する。
- **キャラクターシート自動分割**: 1枚のシート画像から被写体パネルを自動検出
  (背景/アルファの連結成分解析)し、検出パネルをUI上でユーザーが各ビューに
  割り当てて生成に使用できる。自動割当の初期推定は左からの並び順ヒューリスティクス
  (正面→側面→背面の順を仮定)とし、必ずUIで修正可能とする。

### 3.9 テクスチャ生成 (FR-10)【Phase 3】
- `texture_mode: none|paint` を追加。`paint` 時は Hunyuan3D-2 paint パイプライン
  (texgen)で全周テクスチャを生成する。
- ビューア用GLBはテクスチャ付きで出力。4色プリント用にはテクスチャ色を
  頂点カラーにサンプリングした上で従来のパレット量子化・色分割3MF出力(FR-8)を適用する。
- texgenのビルド・依存が利用不可の環境では、機能をUI上で無効表示し、
  正面/背面投影方式(FR-8)にフォールバックする。

### 3.10 プリセット (FR-11)【Phase 3】
- パラメータ一括設定のプリセットを提供: `フィギュア`(高さ100mm/384/20万面)、
  `小型フィギュア`(高さ60mm/256/10万面)、`ペンダント`(高さ40mm/256/8万面/カラーOFF)、
  `高精細`(高さ150mm/512/40万面)。選択後も個別調整可能。

### 3.11 サポート必要箇所ヒートマップ (FR-12)【Phase 3】
- ビューアの表示モードに「オーバーハング」を追加。造形方向(Z)に対する面の傾斜角から
  サポートが必要になりやすい箇所(既定45°超)を赤系グラデーションで強調表示する。
  クライアントサイド(Three.js)のみで計算し、閾値スライダーで調整できる。

### 3.12 ジョブ履歴 (FR-7)
- セッション中に生成したジョブの一覧(サムネイル・状態・作成日時)を表示し、
  過去の生成結果をビューアで再表示・再ダウンロードできる。
- ジョブデータはサーバ側 `data/jobs/<job_id>/` に永続化(入力画像・メッシュ・メタデータJSON)。

## 4. 非機能要件

| ID | 要件 |
|---|---|
| NFR-1 | 生成時間: Hunyuan3D-2使用時、1モデルあたり60秒以内を目標(本GPU環境) |
| NFR-2 | 同時実行: 生成ジョブはGPUメモリ保護のため直列実行(キュー処理)。 |
| NFR-3 | モデルのロードは初回リクエスト時に1度だけ行い、以降常駐させる。 |
| NFR-4 | ローカル利用前提のため認証は不要。バインドは127.0.0.1をデフォルトとする。 |
| NFR-5 | mockジェネレータによりGPU/モデル無しでも全API・UIがE2Eで動作すること。 |
| NFR-6 | VRAM最小要件(既定パラメータでの実測ピーク+余裕): 形状生成のみ 16GB(実測ピーク約12GB)、テクスチャ生成(FR-10)併用時 32GB(実測ピーク約25GB)。mockはGPU不要。詳細はREADME「VRAM最小要件」参照。 |

## 5. API仕様(REST)

ベースURL: `http://127.0.0.1:8000`

| メソッド | パス | 説明 |
|---|---|---|
| POST | `/api/jobs` | multipart: `image`(必須), `params`(JSON文字列, 任意)。ジョブ作成→ `{job_id}` |
| GET | `/api/jobs` | ジョブ一覧 |
| GET | `/api/jobs/{id}` | ジョブ状態・メタデータ(進捗、メッシュ統計含む) |
| GET | `/api/jobs/{id}/input` | 入力画像(背景除去後があればそれ) |
| GET | `/api/jobs/{id}/model.glb` | ビューア表示用GLB |
| GET | `/api/jobs/{id}/download?format=stl\|3mf\|obj\|glb` | エクスポート |
| DELETE | `/api/jobs/{id}` | ジョブ削除 |
| GET | `/api/health` | ヘルスチェック(使用ジェネレータ名・GPU情報含む) |
| GET | `/` | フロントエンドUI |

ジョブJSONスキーマ(抜粋):
```json
{
  "job_id": "uuid",
  "status": "queued|preprocessing|generating|postprocessing|completed|failed",
  "created_at": "ISO8601",
  "error": null,
  "params": {"steps": 30, "guidance_scale": 5.5, "octree_resolution": 384,
              "seed": 1234, "remove_bg": true, "target_height_mm": 100,
              "max_faces": 200000, "color_mode": "none|color4", "n_colors": 4},
  "stats": {"vertices": 0, "faces": 0, "watertight": true,
             "bbox_mm": [0,0,0], "volume_cm3": 0.0,
             "palette": [{"hex": "#aabbcc", "face_ratio": 0.25}]}
}
```

## 6. 画面仕様

シングルページ構成(左右2ペイン):
- **左ペイン**: 画像アップロード(D&D)、パラメータ設定フォーム、生成ボタン、
  進捗表示、ジョブ履歴リスト。
- **右ペイン**: Three.js 3Dビューア(全高)、上部にモデル情報バー、
  下部にエクスポートボタン群(STL / 3MF / GLB / OBJ)。
- 日本語UI。

## 7. 制約・前提

- キャラクターシートからの厳密なマルチビュー再構成(正面+側面+背面の整合)は
  本バージョンのスコープ外。単一画像からの生成品質で対応する。
- テクスチャ(カラー3Dプリント)は将来拡張(Hunyuan3D-2 paint pipeline)。
- サポート材生成・スライスはスライサー(Cura等)の役割とし、本アプリは行わない。
